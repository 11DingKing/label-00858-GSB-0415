# 星际空间可视化

## 1 如何运行

### 本地开发
前置条件：Node.js (v16+)

1.  进入项目目录：
    ```bash
    cd trea-space-visualizer
    ```
2.  安装依赖：
    ```bash
    npm install
    ```
3.  启动开发服务器：
    ```bash
    npm run dev
    ```
4.  在浏览器中打开 `http://localhost:5173`，根据提示允许摄像头访问。

### Docker 生产环境构建
前置条件：Docker, Docker Compose

1.  构建并运行容器：
    ```bash
    docker-compose up --build
    ```
2.  在浏览器中打开 `http://localhost:8080`。

## 2 服务组件

*   **前端**：原生 ES Modules + Vite（端口 8080/5173）
*   **渲染引擎**：Three.js WebGL 渲染器
*   **计算机视觉**：MediaPipe Hands（通过 CDN 加载）

## 3 测试说明

*（本应用无需登录认证）*

*   **测试环境**：支持摄像头的标准 Web 浏览器（Chrome/Safari/Firefox/Edge）
*   **操作说明**：
    *   **旋转**：举起手，左右旋转手掌。
    *   **拉远**：握拳 ✊ 并快速将手从摄像头拉远。
    *   **拉近**：张开手掌 🖐️ 并将手靠近摄像头。

## 4 题目内容

create a webapp using vanilla is, html, css, modernthreeis, mediapipe. 如果mediapipe 来自CDN，要用常规的<script> 方式引入变量。画面中是由大量球形小的粒子构成的一个绚烂的橙色的星球，星球球面座多能金现之济 实能在求面容寒了更多的粒子）。周围环绕着日色的粒子，构成星球的两层环绕星环。粒子要很大很亮。7360摄像头的中心一直对着星球的中心，可以通过用户的手势控制摄像头，用户旋转手的时候，摄像头会随之旋转，就像星球在同步的顺时针或逆时针旋转；当握拳并拉远摄像 62头的时候，相机会较快远离星球，速度先慢后快，而当5指张开并靠近摄像头的时候，摄像头会快速靠近星球，速度先快后慢（摄像头距离星球的中心有个最小距离）。9


**视觉效果**：
- 由大量球形粒子构成的绚烂橙色星球
- 两层白色粒子环绕星环
- 粒子要很大很亮
- 摄像头中心始终对准星球中心

**交互控制**：
- 通过手势控制摄像头
- **旋转**：旋转手掌 -> 摄像头绕星球旋转
- **拉远**：握拳 + 拉远 -> 摄像头远离星球（速度：先慢后快）
- **拉近**：张开手掌 + 靠近 -> 摄像头靠近星球（速度：先快后慢）
- 摄像头与星球中心有最小距离限制

## 项目结构

```
trea-space-visualizer/
├── src/
│   ├── core/           # 应用逻辑、渲染器、相机控制器
│   ├── world/          # 3D 对象（星球、星环）
│   ├── input/          # MediaPipe 手势集成
│   ├── ui/             # UI 覆盖层管理器
│   └── config.js       # 全局配置
├── public/             # 静态资源
├── Dockerfile          # 多阶段多架构构建
├── docker-compose.yml  # 容器编排
└── index.html          # 入口文件（含 CDN 脚本）
```

## 功能特性

*   **视觉层次**：深空背景上的毛玻璃风格 UI 覆盖层
*   **组件架构**：逻辑、渲染、输入清晰分离
*   **稳健输入**：基于状态机的手势识别，处理摄像头噪声数据
*   **工程化**：基于 Vite 的构建系统，Docker 容器化部署
