# Three.js + MediaPipe 3D粒子交互项目架构分析

## 一、渲染管线组织与调用链

### 1. 初始化流程

```
DOMContentLoaded → new App() → App.start()
```

- **App 构造器初始化**：
  1. 创建 Renderer 实例（负责 Three.js 场景、相机、渲染器初始化）
  2. 创建 UIManager 实例（负责UI状态管理）
  3. 创建 CameraController 实例（负责相机运动控制）
  4. 创建 Planet 实例（核心粒子星球）
  5. 创建 Rings 实例（环绕粒子环）
  6. 创建 GestureManager 实例（负责 MediaPipe 手势识别）
  7. 初始化场景光照
  8. 创建时钟用于动画计时

### 2. 每帧更新调用链

```
requestAnimationFrame → App.animate() →
  1. 获取当前时间 this.clock.getElapsedTime()
  2. CameraController.update() → 处理相机平滑运动、约束、UI更新
  3. Planet.update(time) → 星球粒子自转
  4. Rings.update(time) → 粒子环自转（反向）
  5. Renderer.render() → 执行 Three.js 渲染
```

## 二、GestureManager 交互与数据传递流程

### 1. 初始化阶段

GestureManager 初始化时接收 CameraController 和 UIManager 实例作为依赖：

```
App构造器 → new GestureManager(cameraController, uiManager)
```

- 内部初始化 MediaPipe Hands 模型和摄像头
- 注册 onResults 回调处理每一帧手势识别结果

### 2. 手势数据流向

```
MediaPipe 摄像头帧 → hands.send() → onResults(results) →
  1. 手势检测：
     - 计算手的旋转角度 → 调用 cameraController.rotate()
     - 检测握拳/张手手势 → 调用 cameraController.startZoomOut()/startZoomIn()/stopZoom()
  2. UI同步：
     - 更新置信度显示
     - 高亮对应功能卡片
     - 更新交互强度显示
```

### 3. 与场景对象的交互特点

- **无直接耦合**：GestureManager 不直接操作 Three.js 场景或粒子对象，所有场景修改都通过 CameraController 间接完成
- **关注点分离**：手势识别层只负责解析手势语义（旋转、缩放），不关心相机具体如何运动、场景如何渲染
- **事件驱动**：手势识别结果通过方法调用传递给 CameraController，不需要主动轮询

## 三、Planet.js 与 Rings.js 职责划分与耦合分析

### 1. Planet.js 职责

- 核心粒子星球的创建与管理
- 生成球面分布的金色/橙色发光粒子
- 70%粒子在表面，30%在内部，形成更真实的球体效果
- 实现星球自转动画（y轴正方向旋转，速度 0.05 rad/单位时间）

### 2. Rings.js 职责

- 环绕星球的双层粒子环创建与管理
- 生成白色/蓝色调的环形分布粒子
- 分为内环和外环两层，有厚度效果
- 实现环带自转动画（y轴负方向旋转，速度 0.02 rad/单位时间）
- 自带倾斜角度，增强立体感

### 3. 耦合关系分析

- **无直接耦合**：两个模块完全独立，没有互相引用或调用
- **相同依赖**：都依赖 THREE.js 和 CONFIG 配置，都接收 scene 实例作为构造参数
- **职责完全分离**：
  - Planet 负责中心球体粒子系统
  - Rings 负责外围环带粒子系统
  - 两者只在 App.animate() 中被独立调用 update 方法
- **松耦合设计**：可以单独替换/移除其中任意一个模块，不影响另一个的运行

## 四、整体架构特点

1. **模块化设计**：各模块职责单一，边界清晰
2. **依赖注入**：App 作为入口统一创建和注入依赖，降低模块间耦合
3. **分层架构**：
   - 输入层：GestureManager 处理手势识别
   - 控制层：CameraController 处理相机运动
   - 渲染层：Renderer 负责底层渲染
   - 场景层：Planet/Rings 负责场景内容
   - UI层：UIManager 负责界面展示
4. **单向数据流**：手势数据 → 控制器 → 场景更新 → 渲染，没有循环依赖
